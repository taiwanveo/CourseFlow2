# CourseFlow v2 關鍵程式碼清單

本清單不是列出「檔案最多」的地方，而是列出「最直接控制系統核心行為」的模組。
挑選標準如下：

1. 直接決定文稿是否進佇列、是否能切到 Worker 執行。
2. 直接控制 WVP dist 的保存、還原、錄製與 MP4 輸出。
3. 直接控制背景任務狀態、進度回寫與 Storage 上傳。
4. 一旦出錯，會明顯影響部署、匯出、預覽或正式環境穩定性。

## 依功能分區版本

下面這一版是給「從產品流程反推程式碼」時使用的地圖。
如果你是依功能接手，而不是依底層模組接手，先看這一節會比較快。

### 登入 / 權限

- `apps/web/src/middleware.ts`
	用途：站在所有受保護頁面的最前面，根據 Supabase session 決定要放行、導向 `/login`，或把已登入使用者從登入頁導回應用內。
- `apps/web/src/lib/supabase/admin.ts`
	用途：建立 server-side service client，提供 API route 與後端邏輯存取資料庫、Storage、背景任務資料。
- `apps/web/src/app/login/**`
	用途：登入頁與登入相關 UI；屬於入口層，不是底層決策邏輯，但它是使用者進入系統的第一站。

### 文稿 / Composition

- `apps/web/src/app/api/projects/[id]/generate-content/route.ts`
	用途：從文章或輸入內容生成文稿、章節、步驟與螢幕內容，是文稿階段最直接的 API 入口。
- `apps/web/src/app/api/projects/[id]/composition/route.ts`
	用途：更新專案 composition，是文稿資料寫回資料庫的核心入口。
- `packages/composition/src/**`
	用途：承載 composition 的型別與結構定義，決定文稿資料能長成什麼樣子。
- `packages/llm/src/prompts.ts`
	用途：定義文稿與螢幕內容相關提示詞，影響內容生成品質與結構。

### Craft / 視覺動效

- `apps/web/src/lib/wvp-chapter-craft.ts`
	用途：章節 Craft 主控邏輯，負責單章生成、批次 Craft、套模板、試執行第 1 章等核心流程。
- `apps/web/src/app/api/projects/[id]/wvp/trial-chapter-1/route.ts`
	用途：觸發第 1 章試跑，是使用者校準風格錨點的主要入口。
- `apps/web/src/app/api/projects/[id]/wvp/batch-craft/route.ts`
	用途：批次生成所有章節 Craft，通常是視覺階段最重的一條主路徑。
- `apps/web/src/lib/wvp-presentation-sync.ts`
	用途：把 Craft 結果同步成真正的 presentation 工作區，並進一步 build 成可預覽的 dist。
- `apps/web/src/lib/wvp-craft-illustrations.ts`
	用途：管理 Craft 階段配圖工作室的素材落地與關聯。
- `apps/web/src/lib/wvp-illustration-sync.ts`
	用途：把步驟配圖同步進 presentation 工作區，讓最終章節模板能真正引用圖片。

### 語音 / Audio

- `apps/web/src/app/api/projects/[id]/synthesize-audio/route.ts`
	用途：語音生成 API 入口，決定是走 Worker 佇列還是 Web inline。
- `apps/web/src/lib/run-synthesize-audio.ts`
	用途：Web inline 模式下的 TTS 執行器，負責實際合成音訊與回寫結果。
- `apps/web/src/lib/job-queue.ts`
	用途：判斷語音任務是否應進 BullMQ，是語音流程是否切到 Worker 的分流開關。
- `apps/worker/src/processors.ts`
	用途：Worker 端實際執行 TTS 任務，回寫 `job_runs` 與音訊檔案，是正式背景語音流程的核心。
- `packages/tts/src/**`
	用途：封裝各種 TTS provider 的呼叫介面，決定語音合成底層如何被執行。

### 匯出 / Build / MP4

- `apps/web/src/app/api/projects/[id]/wvp/build/route.ts`
	用途：觸發完整 WVP build，將 Craft / 音訊 / 素材整合為可預覽的 dist。
- `apps/web/src/app/api/projects/[id]/export/route.ts`
	用途：MP4 匯出主入口，決定是否可匯出、是否進 Worker，以及是否使用 WVP 匯出路徑。
- `apps/web/src/lib/wvp-dist-storage.ts`
	用途：保存與還原 dist，讓 build 成果能跨容器與跨機器重用。
- `apps/web/src/lib/run-wvp-render.ts`
	用途：Web inline MP4 匯出主控流程。
- `apps/worker/src/process-render-wvp.ts`
	用途：Worker 背景 MP4 匯出主控流程。
- `apps/worker/src/processors.ts`
	用途：管理 render job 狀態、進度、成功失敗回寫，是背景匯出的總協調器。
- `packages/presentation/src/record.ts`
	用途：真正把 WVP dist 播放並錄成影片的底層錄製器。

### 預覽 / Dist / Storage

- `apps/web/src/lib/wvp-presentation-sync.ts`
	用途：建立與更新每個專案的 `presentation/` 工作區，並 build 出 `dist/`。
- `apps/web/src/lib/wvp-dist-storage.ts`
	用途：將 `dist/` 快取到 Supabase Storage，支援 Render / Worker 還原。
- `apps/web/src/lib/wvp-export.ts`
	用途：檢查目前是否已具備可匯出的 WVP dist，是匯出前的 readiness gate。
- `packages/presentation/src/build.ts`
	用途：實際執行 Vite build，把工作區輸出成靜態預覽檔。

### Worker / 背景任務

- `apps/worker/src/index.ts`
	用途：Worker 行程入口，負責啟動佇列消費與心跳機制。
- `apps/worker/src/processors.ts`
	用途：Worker 任務總處理器，涵蓋 TTS、傳統 render、WVP render。
- `packages/shared/src/queue-names.ts`
	用途：定義所有 queue 名稱與前綴，避免 v1 / v2 任務互相污染。
- `apps/web/src/lib/job-queue.ts`
	用途：從 Web 端決定哪些任務要切進 Worker，是背景任務體系的進入點。

## 依功能閱讀順序

如果你是按產品流程讀程式，建議順序如下：

1. 登入：先看 `apps/web/src/middleware.ts`，理解哪些頁面需要 session。
2. 文稿：看 `generate-content/route.ts`、`composition/route.ts`、`packages/composition`。
3. Craft：看 `wvp-chapter-craft.ts` → `trial-chapter-1/route.ts` → `batch-craft/route.ts`。
4. 語音：看 `synthesize-audio/route.ts` → `job-queue.ts` → `run-synthesize-audio.ts` / `apps/worker/src/processors.ts`。
5. 匯出：看 `wvp/build/route.ts` → `wvp-dist-storage.ts` → `export/route.ts` → `run-wvp-render.ts` / `process-render-wvp.ts` → `packages/presentation/src/record.ts`。

## 1. apps/web/src/lib/job-queue.ts

- 檔案用途：決定 Web 端是否把 TTS / Render 任務送進 BullMQ，或在本機 / 單機模式下直接 inline 執行。
- 為什麼關鍵：這是 Web 與 Worker 的分流開關；若判斷錯誤，正式環境可能任務不入隊，本機也可能誤以為 Worker 在線。
- 上游依賴：環境變數 `COURSEFLOW_INLINE_JOBS`、`COURSEFLOW_FORCE_QUEUE`、`REDIS_URL`，以及 `@courseflow/shared` 的 Redis heartbeat 判斷。
- 下游影響：所有需要背景任務的 API 路徑，特別是語音生成與影片匯出。

## 2. apps/web/src/lib/wvp-dist-storage.ts

- 檔案用途：把 `presentation/dist` 上傳到 Supabase Storage，或在 Render / 易失環境下從 Storage 還原回本地。
- 為什麼關鍵：WVP 預覽與 MP4 匯出都依賴 dist；如果這裡壞掉，預覽、跨機器還原、Worker 錄製都會失效。
- 上游依賴：`@courseflow/presentation` 內的 dist manifest / upload / download helpers，以及本機 `presentationDistDir()`。
- 下游影響：WVP 預覽頁、Storage 快取命中率、背景錄製是否能取得完整靜態檔。

## 3. apps/web/src/lib/run-wvp-render.ts

- 檔案用途：在 Web 進程中執行 WVP MP4 匯出，優先使用本機已 build 的 dist，否則回退到 Storage 快取，再呼叫 Playwright 錄製。
- 為什麼關鍵：這是 inline 匯出路徑的主協調器；本機開發、沒有 Worker 的 fallback、除錯單機流程都會走這裡。
- 上游依賴：本機 dist 是否存在、Storage 是否已有 WVP dist、`@courseflow/presentation/record` 是否能成功錄製。
- 下游影響：Render Job 輸出 MP4 是否能落到 Supabase Storage。

## 4. apps/worker/src/process-render-wvp.ts

- 檔案用途：Worker 版本的 WVP 匯出流程，從 Storage 還原 dist，交給 Playwright 錄製，最後把 MP4 回寫 Storage。
- 為什麼關鍵：正式環境若走背景匯出，這支就是真正產出 MP4 的主路徑。
- 上游依賴：Storage 內的 dist manifest 與 dist 檔案、`@courseflow/presentation/record`、Supabase service role。
- 下游影響：`render_jobs.output_path` 對應的影片檔是否存在且可下載。

## 5. apps/worker/src/processors.ts

- 檔案用途：Worker 的總入口，處理 TTS 合成、傳統 HyperFrames render、WVP render，並把進度 / 完成 / 失敗狀態回寫資料庫。
- 為什麼關鍵：這裡同時控制 job 生命周期、Storage 資產下載、音訊合成、影片渲染與錯誤處理，是 Worker 執行面的總協調器。
- 上游依賴：Supabase、TTS provider、HyperFrames、WVP render processor、composition snapshot。
- 下游影響：`job_runs`、`render_jobs`、Storage 音訊檔、Storage MP4 檔、使用者在 Studio 看到的任務狀態。

## 6. packages/presentation/src/record.ts

- 檔案用途：對已 build 完的 WVP dist 啟動本機靜態站，用 Playwright 自動播放錄屏，再用 ffmpeg 轉成 MP4。
- 為什麼關鍵：不論是 Web inline 匯出還是 Worker 匯出，最後真正把互動式 WVP 轉成影片的關鍵步驟都在這裡。
- 上游依賴：dist 完整性、靜態站可被正確載入、Playwright Chromium、ffmpeg、前端自動播放完成旗標。
- 下游影響：MP4 是否能成功產生、錯誤是否能正確反映為「dist 缺檔／自動播放未啟動／錄製逾時」。

## 關鍵程式碼閱讀順序

若你是第一次接手，建議依下列順序閱讀：

1. `apps/web/src/lib/job-queue.ts`：先理解什麼情況會走 inline，什麼情況會進 Worker。
2. `apps/web/src/lib/wvp-dist-storage.ts`：再理解 dist 如何跨機器保存與還原。
3. `apps/web/src/lib/run-wvp-render.ts`：理解 Web 端單機匯出主流程。
4. `apps/worker/src/process-render-wvp.ts`：理解正式背景錄製主流程。
5. `apps/worker/src/processors.ts`：理解 Worker 的任務生命週期與錯誤回寫。
6. `packages/presentation/src/record.ts`：最後看底層錄製實作，理解真正產生 MP4 的地方。

## 不列入本清單，但仍重要的相鄰模組

- `apps/web/src/lib/wvp-presentation-sync.ts`：負責建立 / build presentation 工作區，但更偏內容同步與打包，不是最終執行面主控。
- `packages/presentation/src/build.ts`：負責 Vite build，本身重要，但在系統層級上仍受上列模組協調。
- `apps/web/src/app/api/projects/[id]/**`：負責路由與權限，但大多數屬於進入點，真正決策仍在上述 lib / worker 模組。
