# 教學圖片產生器 PRD v1

## 1. 文件資訊
- 文件版本：v1.0
- 日期：2026-06-01
- 產品名稱：教學圖片產生器
- 平台：Windows 桌面（Electron Portable）

## 2. 產品願景
讓一般使用者在 Windows 上「雙擊即可用」地把教材內容快速轉換成教學圖片，支援 BananaX 風格與多 provider 模型切換，並輸出 PNG/JPG。

## 3. 目標與非目標
### 3.1 目標
1. 輸入教材內容後可一鍵生成教學圖片。
2. 可在兩層式 BananaX 介面中完成風格搜尋與套用。
3. 支援至少 4 家 provider，且只顯示可生圖模型。
4. 提供尺寸預設與自訂寬高。
5. 支援 PNG/JPG 輸出。
6. 透明背景僅在模型原生支援時可用。
7. API Key 存於 Windows Credential Manager。

### 3.2 非目標
1. 不支援 PDF 輸出。
2. 不做自動去背。
3. 不做 Supabase 帳號/認證/雲端儲存。
4. 不做 CourseFlow 的 chapter/craft/sync 產品流程。
5. 不做影片、口播、動畫流程。

## 4. 目標使用者
1. 教師與講師。
2. 課程企劃與教材編輯。
3. 知識型內容創作者。

## 5. 使用者故事
1. 作為講師，我希望貼上教材內容後快速得到可用教學配圖。
2. 作為內容編輯，我希望能搜尋 BananaX 風格並看大圖細節再決定套用。
3. 作為進階使用者，我希望切換不同 provider 與模型比較結果。
4. 作為一般使用者，我希望不需要安裝依賴，雙擊就可用。

## 6. 主要流程
1. 啟動應用。
2. 輸入教材內容與補充指令。
3. 開啟 BananaX 第一層頁面，搜尋/過濾/選風格。
4. 進入第二層頁面，查看大圖與提示詞，套用。
5. 選擇 provider 與模型（僅顯示可生圖模型）。
6. 選尺寸（預設或自訂）與輸出格式（PNG/JPG）。
7. 若模型支援透明，可勾選透明輸出。
8. 生成並下載結果。

## 7. 功能需求（FR）
### FR-01 教材輸入
- 支援文字輸入（多行）。
- 支援附加提示詞。

### FR-02 BananaX 兩層式瀏覽
- 第一層：搜尋、分類、過濾、中尺寸預覽列表。
- 第二層：大圖預覽、風格說明、完整提示詞、套用操作。

### FR-03 Provider 管理
- 至少 4 家 provider（OpenAI、OpenRouter、Gemini、Together）。
- API Key 新增/更新/刪除。
- 金鑰寫入 Windows Credential Manager。

### FR-04 模型清單與過濾
- 依 provider 拉取最新模型清單。
- 僅顯示 imageGeneration 模型。
- 可顯示能力標記（透明、自訂尺寸）。

### FR-05 圖片生成
- 組合教材摘要 + 使用者提示 + BananaX 風格。
- 呼叫選定 provider/model 生成圖片。

### FR-06 輸出
- 格式：PNG/JPG。
- 尺寸：預設 + 自訂寬高。
- 透明：僅在模型支援時可啟用。

### FR-07 歷史記錄
- 記錄最終 prompt、provider/model、尺寸、格式、透明狀態。
- 可重新打開歷史結果並下載。

## 8. 非功能需求（NFR）
1. 啟動：一般機器可在合理時間完成啟動。
2. 可用性：首次使用者可在 3 分鐘內完成首張生圖。
3. 穩定性：模型 API 失敗需提供可理解錯誤。
4. 安全：API Key 不落地於明文檔案或 SQLite。
5. 維護性：Provider 接入透過標準化 Adapter，不與 UI 強耦合。

## 10. 透明背景策略
1. UI 顯示「透明輸出」開關。
2. 僅當模型 capability 標記支援透明時開啟。
3. 若不支援，明確禁用並提示原因。
4. 不做去背 fallback。

## 11. 成功指標
1. 首次安裝到首張圖片成功率。
2. 模型清單刷新成功率。
3. 圖片生成成功率。
4. 風格選擇完成率（第一層到第二層再套用）。

## 12. 驗收標準（UAT）
1. 可雙擊啟動 Portable 應用。
2. 可儲存並讀取 4 家 provider API Key。
3. 可刷新每家 provider 模型清單。
4. 下拉清單僅顯示可生圖模型。
5. 可完成 PNG/JPG 輸出。
6. 透明僅在支援模型可選。
7. BananaX 兩層頁可完成搜尋與套用。

## 13. 風險與限制
1. 各 provider 模型欄位差異大，需本地規則覆寫。
2. 部分 provider 能力標記不完整，需 allowlist/denylist 補正。
3. 透明能力可能隨模型版本變動，需週期更新能力規則。

## 14. 版本規劃
- M0：骨架與流程打通（單 provider + 本地檔案）。
- M1：4 provider + BananaX 兩層頁 + PNG/JPG。
- M2：穩定化（更多測試、快取治理、錯誤處理強化）。
