# 錄製與後期合成

網頁做完 + 音訊合成完之後，**Auto 模式 + 螢幕錄製可以一鏡到底**——
不需要手動點選推進 step、也不需要後期對音訊。

如果你不打音訊，仍可走"手動點選 + 後期配"的傳統路徑（在檔案末尾）。

---

## 推薦流程：Auto 模式一鏡到底

### 前置

- 章節程式碼做完，每章都有 `narrations.ts`
- 已經跑過 `npm run extract-narrations` + `npm run synthesize-audio`，
  `public/audio/<id>/<step>.mp3` 全部就位
- `npm run dev` 跑著，瀏覽器能開啟頁面

### 錄製步驟

1. **瀏覽器全屏**（F11 / Ctrl+Cmd+F），URL 改成
   `http://localhost:5173/?auto=1`
2. 看到 "Press SPACE to start" 蒙層 = Auto 模式就緒
3. **開啟螢幕錄製**（QuickTime / OBS / Cmd+Shift+5），開始錄
4. **按一次 Space** → 蒙層消失 → step 0 出現，1.mp3 自動播 →
   播完自動推進到 step 1 → 2.mp3 → … → 最後一個 step 播完 → 停在終態
5. **停止錄製** → 後期裁掉頭尾（Space 那一下、最後停在終態的尾巴）就是
   成品

整個過程**完全不用點滑鼠**。音影片天然同步，不需要後期對軌。

> **Auto 模式嚴格按音訊結束推進**（+ 200ms 緩衝），沒有"等動畫跑完"
> 的兜底。如果你看到某步動畫被切了一半 → 說明該 step 動畫長於口播，
> 回章節程式碼改：寫更長口播 / 拆 step / 調動畫速度。

### 錄屏工具

| 平臺 | 工具 | 設定 |
|---|---|---|
| macOS | Cmd+Shift+5 → 錄製選定視窗 | 選瀏覽器視窗；瀏覽器全屏後輸出就是 1920×1080 |
| macOS | QuickTime → 檔案 → 新建螢幕錄製 | 同上 |
| 跨平臺 | OBS Studio | 視窗捕獲，Canvas 1920×1080，60fps |

### 模式速查

| URL / 快捷鍵 | 行為 |
|---|---|
| 直接開啟（預設） | Manual：點選 / ←→ 推進，不播音訊 |
| `?audio=1` 或按 `M` | Audio：進入 step 自動播音訊，但**手動點滑鼠推進** |
| `?audio=1` + 再按 `M` | Auto：進入 step 自動播 + 自動推進（錄製用） |
| Auto 模式下首次按 `Space` | 啟動 Auto 播放（繞過瀏覽器自動播放限制） |

也可以滑鼠移到右上角，會出現一個隱藏的模式切換按鈕。

## 備用流程：沒合成音訊時手動錄屏

如果你跳過了音訊合成（`Checkpoint Audio` 選了"不合成"），按老方法：

1. 瀏覽器全屏 → 開啟 `localhost:5173`（預設 Manual 模式）
2. **重新整理一次**清空歷史 step
3. 開始錄屏 → 按口播節奏點選空白推進 step
4. 後期用任何剪輯軟體配音 + 調時間線

### 後期工具

| 工具 | 適合 |
|---|---|
| **DaVinci Resolve** | 跨平臺免費、能處理多段音訊拼接 |
| **iMovie** | macOS 簡單場景 |
| **CapCut / 剪映** | B 站 / 抖音風加字幕 |
