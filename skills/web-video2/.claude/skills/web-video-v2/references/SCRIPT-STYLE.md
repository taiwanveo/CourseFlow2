# 文章 → 口播稿風格指南

把書面文章轉成"說出來不彆扭"的口播稿，預設 **B 站風格**。其他平臺風格
變體見末尾。

> **兩條底線**（任一不過，整稿重寫）：
>
> 1. **資訊保留度 ≥ 60%**（詳見下一節）—— 口播稿是"換說法"，不是
>    "摘要"。刪冗餘 / 修飾可以，刪事實 / 資料 / 案例 / 論證鏈不行。
> 2. **去 AI 味** —— 即使句子已經短句化、口語化、第二人稱，AI 的腔調
>    （假共情 / 假深刻 / 自我標榜 / 萬能模板 / 排比堆砌）念出來仍然
>    一耳朵是機器。**口播比文字更怕 AI 味**。
>
> 8 條原則講**形式**，「去 AI 味五類」講**風骨**。兩者都建立在
> 「資訊保留度」這一最高約束之上。

---

## 最高原則：資訊保留度 ≥ 60%（刪減不得超過 40%）

> 這條**優先於**下面所有 8 條原則和去 AI 味五類。先保證資訊沒丟，
> 再調形式 / 風骨。

**判定標準**：

- `len(script.md) ÷ len(article.md) ≥ 0.6`（中文按字元數計；程式碼塊 /
  Markdown 標記 / 引用源不計入）
- 關鍵事實（數字 / 名稱 / 時間 / 論斷）**逐項對得上**，不能整段消失
- 文章裡的**論證鏈**（前提 → 結論 / 因果 / 對比）必須保留，可以換說
  法但不能跳步

**允許大幅壓縮的**：套話開場 / 重複總結 / 學術性鋪墊 / 引文出處的
完整書目格式 / 客套結語。

**禁止壓縮的**：

- 關鍵資料和數字（哪怕"翻譯成感受"也要保留原始數字本身作旁註）
- 具體案例 / 例子 / 故事
- 多步論證裡的中間步驟
- 反方觀點 / 限制條件 / 例外情況（最容易被 AI 一刀切掉）

**如果原文確實太長**（比如 5000+ 字 article 做 3 分鐘影片）→ 不要靠
壓縮硬塞，**告訴使用者**："原文 X 字，按 60% 留存最少需要 ~Y 分鐘影片，
要麼拉長影片要麼拆成多集"。**不要**自作主張縮到 30% 然後假裝做完了。

## B 站風格的 8 條原則（形式層面）

### 1. 口語化：所有書面語換成"說出來不會彆扭"的句子

| 書面語 | 口播 |
|---|---|
| 綜上所述 | 所以你看 |
| 具有非凡的意義 | 這件事真的挺重要 |
| 在……方面有顯著提升 | 在……上明顯變強了 |
| 該工具旨在 | 這玩意是用來 |
| 筆者認為 | 我覺得 |
| 由此可見 | 你看吧 / 這就是為啥 |
| 與之相反 | 反過來 |
| 據相關資料顯示 | 我查了下 |
| 詳情請參閱 | 想看細節自己去翻 |

判斷標準：**把它念出來，會不會覺得"這人怎麼這麼端著"**？會就改。

### 2. 短句：每句 ≤ 20 字

長句拆短句，逗號優先於複合句。

| 原文 | 改寫 |
|---|---|
| "儘管它在文字渲染方面有了顯著的提升，但在某些複雜場景下仍然會出現錯誤。" | "文字渲染確實變強了。但複雜場景下，它還是會翻車。" |
| "我們透過對比測試發現該模型在指令遵循能力上明顯優於上一代。" | "我做了一組對比測試。指令遵循這一項，比上一代強太多了。" |

講話的人不會喘不上氣。螢幕前的人也不會。

### 3. 第二人稱：多用"你 / 我們"，少用"使用者 / 讀者"

| 原文 | 改寫 |
|---|---|
| 使用者可以透過該介面實現 | 你只要調一下這個介面 |
| 讀者將會發現 | 你會發現 |
| 開發者需要注意 | 你寫的時候要小心 |
| 本文將介紹 | 我們來看一下 |

讓螢幕前的人感覺**你在跟他說話**，而不是宣讀公告。

### 4. 開頭有鉤子（cold open）

第一句必須**抓住人**。三種通用鉤子：

- **懸念**："這是真的嗎？" / "你猜怎麼著" / "我以為是 bug，結果……"
- **反差**："這玩意以前要 3 小時，現在 30 秒。" / "上週還人人吹爆，今天就翻車了。"
- **利益**："看完這條，你就能省下買外掛的錢。" / "學會這一招，開會再也不怕被點名。"

**禁止**（前兩個是 PPT 感，後兩個是 AI 味假共情）：

- ❌"大家好今天我們來講講" 這種自我介紹開場
- ❌"本期影片我們將探討" 這種 PPT 標題感
- ❌"在開始之前，讓我們先了解一下背景" 這種先講一堆鋪墊
- ❌"我知道你最近肯定很糾結 X / 你是不是也有這種感覺" 假共情鉤子
- ❌"接下來我說的，可能會顛覆你的認知" 自我加權鉤子

鉤子要在 **3 秒內**砸到觀眾臉上。親不親是觀眾聽完判斷的，不是你一
開口就宣佈的。

### 5. 節奏停頓：用 `---` 分隔每個完整想法

每講完一個聚焦的想法 = 一個 step 邊界 = 一個 `---`。一行一個 idea，
不堆疊。

```markdown
所以呢，這玩意兒真的挺誇張的。

---

舉個例子。前幾天我讓它畫一張海報，三秒就出。

---

但你知道最離譜的是啥嗎？連裡面的中文字都對了。
```

`---` 之間的內容就是後面 `outline.md` 裡的一個 step。

### 6. 數字翻譯成感受

觀眾聽到具體百分比和大數字會**走神**。除非數字本身就是核心內容
（比如"漲了 20 倍"作為衝擊點），否則翻譯成感受。

| 原文 | 改寫 |
|---|---|
| 準確率提升了 47% | 幾乎快了一倍 |
| 覆蓋 120 個國家 / 地區 | 全世界基本到處都用 |
| 使用者數突破 1.2 億 | 現在已經是國民級應用了 |
| 推理速度提升 3.4 倍 | 比以前快多了，不用等 |

例外：**核心衝擊數字**保留並強調（"前後差了 100 倍"、"3 秒 vs 3 小時"）。

### 7. 不堆結構詞 / 不堆萬能模板

口播稿一旦堆"首先 / 其次 / 最後"或"說白了 / 本質上 / 底層邏輯"，
機器味就直接溢位。

**禁止**（結構詞類）：

- ❌"首先 / 其次 / 最後"
- ❌"接下來我們看下面三點"
- ❌"總結一下，本期影片我們講了什麼什麼"
- ❌"下面進入第二部分"

**禁止**（萬能模板類，AI 高頻指紋 —— 詳見去 AI 味第 4 類）：

- ❌"說白了 ……" / "本質上 ……" / "底層邏輯是 ……"
- ❌"一句話總結 ……" / "歸根結底 ……" / "換句話說 ……"

**做法**：把這些詞融進自然過渡，**或者直接砍掉**。

- "首先，文字渲染" → "先說文字渲染"
- "接下來看第二點" → "再看一個"
- "總結一下" → "所以你看"
- "說白了它就是 X" → "它就是 X"
- "一句話總結：Y" → 直接說 Y

觀眾不需要知道你的影片是幾段式 outline。**結構感從內容節奏來，不是
從指令來**。

### 8. 具體例子優先

**抽象**會讓人走神，**具體**會讓人坐住。

| 抽象 | 具體 |
|---|---|
| 它有很強的能力 | 你給它一張貓的照片，讓它換成賽博朋克風，3 秒就出 |
| 效能優越 | 我那臺 5 年前的舊電腦跑得動 |
| 使用者體驗良好 | 我媽第一次用，5 分鐘就上手了 |
| 應用場景廣泛 | 海報、PPT、表情包、PR 套圖都能整 |

**舉例規則：從"我"出發**。"我做了一個測試"、"我讓它畫"、"我那臺電腦"
比"使用者可以"、"通常情況下"親切 100 倍。

## 去 AI 味：五類必清的腔調（風骨層面）

形式做到了 ≠ 沒 AI 味。下面五類是 AI 寫口播稿時最穩定的"指紋"。
**寫完一遍後專門掃一次，逐條砍掉**。

> 唯一判斷標準：**這話念出來，一個真人會這麼說嗎**？不會就改。

### 一、假共情：開口就演"我懂你"

AI 特別喜歡在開頭先做一輪情緒按摩，再進入正題。但口播一開口就裝親
密 = 油膩；觀眾還不認識你，你就宣佈"我懂你"，聽起來更像銷售。

❌ 典型句式：

- "我知道你最近肯定很糾結 X"
- "你是不是也有這種感覺"
- "今天這條影片，是給那些一直在……的朋友"
- "我能理解你的心情"
- "你的感受是被看見的"

✓ 怎麼改：直接拋事實 / 鉤子（見原則 4）。親不親是觀眾聽完判斷的，
不是你一開口就宣佈的。如果原文確實需要表達共情，用具體的、有資訊
量的方式 ——"這個坑我也踩過"比"我懂你的感受"真實十倍。

### 二、假深刻：用"恰恰 / 反而 / 正是"包裝平庸觀點

AI 很擅長用轉折詞製造"頓悟感"。但仔細一看，去掉這些詞，意思一樣，
甚至更清楚。

❌ 典型句式：

- "你以為它強是因為模型大，恰恰相反 ……"
- "這反而是它真正可怕的地方"
- "正是因為它夠簡單，所以才 ……"
- "你想得太對了"
- "不是因為 X，是因為你太 Y 了"

✓ 怎麼改：去掉轉折包裝直接說結論。**自檢方法**："恰恰 / 反而 / 正
是"那句去掉之後，意思變了嗎？沒變就刪。如果去掉後觀點變得平淡 ——
**它本來就平淡**，刪掉換內容，不要用修辭補救。

### 三、自我標榜：說正事前先給自己加權

AI 在說重點之前喜歡先清嗓子，讓觀眾覺得"接下來這話特別重要"。這
在**口播裡更刺耳** —— 觀眾正在被推銷時間。

❌ 典型句式：

- "我必須很認真地說一句"
- "這件事我得展開講"
- "接下來我說的，可能會顛覆你的認知"
- "這一段你一定要聽完"
- "我講一句可能會讓你冷靜一點的話"
- "不躲不藏不繞不逃，今天就把 X 說透"

✓ 怎麼改：**直接說**。好觀點不需要預熱。擔心觀眾划走，靠節奏（短句
+ 鉤子）把人留住，不靠"加權"。

### 四、萬能模板：一聽就是 AI 寫的口頭禪

這些是 AI 生成口播稿的指紋，出現頻率高到觀眾閉著眼都能猜下一句。

| 位置 | ❌ AI 模板 | ✓ 口播版 |
|---|---|---|
| 開頭 | "說白了 ……" / "本質上 ……" / "底層邏輯是 ……" | 直接說"它就是 X" |
| 過渡 | "一句話總結 ……" / "換句話說 ……" / "歸根結底 ……" | 直接說那句話，不預告 |
| 包裝 | "在某種意義上" / "從某種程度上來講" | 刪掉，沒意義 |
| 結尾 | "以上就是本期內容" / "希望對你有幫助" / "感謝觀看" | 具體 CTA："連結評論區置頂" / "下期見" |
| 互動 | "你想讓我繼續講 X 嗎？" / "評論區告訴我" | 給具體動作："去 GitHub 搜 X" / "評論區聊聊你踩過哪個坑" |

> **注意**：這一類和原則 7「不堆結構詞」有重疊 —— 原則 7 在形式層
> 警告，這裡在風骨層警告，**兩層都要過**。

### 五、排比堆砌：連續三句結構一樣

AI 寫中文時有強迫症式的整齊感。**口播比文字更怕排比** —— 念三個一
樣結構的短句，觀眾腦子直接關機。

**判斷標準**：連續 ≥ 3 個結構相同的短句，且去掉幾個意思不變。

❌ 例子：

- "不躲，不藏，不繞，不逃"
- "它會畫圖，會寫字，會做海報，會做 PPT"
- "你的痛我懂，你的累我看到，你的努力我記得"
- "高效，精準，強大"

✓ 怎麼改：保留最有資訊量的一兩個，其餘砍掉；或換成一個具體例子。

- "它會畫圖，會寫字，會做海報，會做 PPT" →
  "它會畫圖，重點是 —— 你昨天那張貓的賽博朋克版就是它出的"

> 真人講話不會追求每段像對聯。允許粗糙，過渡句就讓它過渡。

## 幾個**不要做**的細節

- ❌ **emoji**：除非使用者明確要求，口播稿裡不要 emoji。emoji 是文字
  彈幕，不是講話
- ❌ **括號補充**："這個模型（一種基於 transformer 的架構）很厲害" ——
  括號裡的話講不出來，要麼併入正文要麼刪
- ❌ **羅列書名號 / 引號**："《XX 報告》指出，XX" —— 改成"我看到一份報告說"
- ❌ **"哈哈哈"等口頭語堆砌**：偶爾一次效果好，每段一個就成抽搐
- ❌ **過度網路梗**：梗有半年保鮮期；半年後看就尷尬。除非主題就是網路梗

## 平臺風格變體

如果使用者指定了別的平臺，按下表調（"去 AI 味五類"對所有平臺都適用）：

| 平臺 | 風格基調 | 單句長度 | 鉤子節奏 | 資訊密度 |
|---|---|---|---|---|
| **B 站**（預設） | 自來熟、有段子、節奏快 | ≤ 20 字 | 3 秒鉤子 | 中（5 秒/idea） |
| **抖音 / 影片號** | 更燃、更短、更衝擊 | ≤ 12 字 | 1 秒鉤子 | 高（3 秒/idea） |
| **YouTube long-form** | 舒緩、詳細、可鋪墊 | ≤ 30 字 | 30 秒鋪墊可以 | 低（10 秒/idea） |
| **知乎影片** | 嚴謹一點、有邏輯感 | ≤ 25 字 | 5 秒可鋪一點背景 | 中（5~7 秒/idea） |
| **小紅書** | 閨蜜分享感、有情緒 | ≤ 15 字 | 直接講結論 | 中（5 秒/idea） |

> **小紅書閨蜜感 ≠ 假共情**。"姐妹們聽我說，這個真的踩雷了" 是閨蜜
> 感（具體 + 有情緒）；"我知道你最近肯定很糾結" 是 AI 假共情（空泛
> + 上來就裝熟）。區別在**有沒有具體場景**。

## 主題型別微調

不同主題型別，有不同的語氣基線：

- **技術 / 教程**：可以稍微"端著"一點，但仍然口語化。多用"我們試一下"
  而不是"假設我們……"
- **測評 / 對比**：要"敢說"。"這個不行"、"那個吹過頭了"。中立測評
  會讓人睡著 —— 但**敢說 ≠ 自我標榜**，"我用了一週覺得不行" 是敢
  說，"我必須很認真地說它不行" 是自我標榜
- **新聞 / 資訊**：節奏要快，金句要硬。開頭第一句必須給最炸的點
- **產品 / 營銷**：忌"吹"。不說"完美"、"顛覆"，說"我用了一週，確實
  順手"
- **文化 / 歷史 / 科普**：可以放慢節奏，多用"你想象一下"開場

## 轉寫完之後做什麼

### 1. 落盤 + 切節拍

落到 `script.md`，原文保留為 `article.md`。每段 ≥ 5 句沒用 `---`
切？切一下 ——`---` 是後面 step 邊界，多切總比少切好（合併步驟比拆
步驟容易）。

> ⚠️ **自檢是硬性流程**：下面的三層自檢（形式 / 風骨 / 念出來）寫完
> `script.md` 後**必須**全部走完 → 修改 → 再繼續。**禁止**寫完直接進
> 入"產出 outline"環節。
>
> **執行方式**（按能力降級）：
>
> 1. **優先 Agent Teams**：開一個獨立 reviewer agent，傳入 `script.md`
>    + 本節三層清單，讓它**逐項核查 + 出結論**（哪幾條 fail + 證據 +
>    改寫建議）。
> 2. **其次 subAgent**：當前 agent 沒 Teams 但能開 subagent，用 subagent
>    走同樣流程。
> 3. **都沒有**：自己**嚴格逐項**核查，特別是「念出來」一定要按字面
>    執行。
>
> 拿到結論後**先按 fail 項改稿子，再產出 outline**。

### 2. 形式層自檢（8 條原則）

- [ ] **資訊保留度 ≥ 60%**（**最高優先順序**，不過整稿重寫）—— 用
      `wc -m script.md` / `wc -m article.md` 算比例；關鍵數字 / 案例 /
      論證鏈逐項對照原文，不能整段消失
- [ ] 沒出現 emoji / 原文書名號《》 / 括號補充 /「據 XX 報告顯示」類
      引文格式（口播念不出來）
- [ ] 每句 ≤ 20 字（B 站基準）？
- [ ] 第二人稱？
- [ ] 開頭 3 秒鉤子？
- [ ] 數字翻譯成感受了？（**核心衝擊數字必須保留原值**）
- [ ] 沒"首先 / 其次 / 最後"結構詞？
- [ ] 都是具體例子，不是抽象描述？

### 3. 風骨層自檢（去 AI 味五類）—— **最重要**

逐條掃一次，**任何一條沒透過就回去改**：

- [ ] 全文還有 "說白了 / 本質上 / 底層邏輯 / 恰恰 / 反而 / 正是因為
      / 歸根結底 / 在某種程度上" 等 **AI 高頻詞**嗎？
- [ ] 開頭第一句是鉤子，**不是**"我知道你 / 你是不是也"假共情？
- [ ] 有 "我必須認真說 / 我得展開說 / 可能會顛覆你的認知 / 這段你一
      定要聽完" 等**自我加權**嗎？
- [ ] 有連續 ≥ 3 句結構一樣的**排比**嗎？唸到第三句你自己煩不煩？
- [ ] 結尾是具體 CTA，**不是** "希望對你有幫助 / 以上就是本期內容
      / 感謝觀看" 套話？
- [ ] 全文是否有用 "恰恰 / 反而 / 正是" 包裝的句子？**去掉這些詞
      意思變了嗎**？沒變就刪。

### 4. 念出來測試（終極標準）

隨便挑 3 段念出來 —— 不要在心裡默讀，**真的張嘴念**。

- 唸到哪句你覺得"這話我說不出口" → 那句是 AI 味
- 唸到哪句你想跳過 → 那段是排比 / 自我標榜
- 唸到哪句你想笑出聲（不是因為內容好笑，是因為油膩）→ 那句是假共情

**改完再念。直到三段念下來都自然為止**。

### 5. 在同一次工作裡產出 outline

`script.md` 落盤 + 自檢透過後，**不要立即停下來等使用者**——按
[`OUTLINE-FORMAT.md`](OUTLINE-FORMAT.md) 在**同一次思考**裡繼續產出
`outline.md`（章節切分 + 每步螢幕內容 + 章節級資訊池），然後再進入
Checkpoint Plan 讓使用者一次對齊 5 件事（稿子 / outline / 主題 / 素材
/ 開發模式）。

> 新版合併為一個（詳見 SKILL.md「Phase 1.2 一次產出」）。理由是 outline
> 不再寫動畫後，它的依賴只剩"稿子切節拍 + article 抽資訊池"，可以
> 和稿子一起做完。
